 Example 139. Word BreakReport Issue

In this article, we'll use the framework to solve Word Break. So far, in this card, this is the most unique and perhaps the most difficult problem to see that dynamic programming is a viable approach. This is because, unlike all of the previous problems, we will not be working with numbers at all. When a question asks, "is it possible to do..." it isn't necessarily a dead giveaway that it should be solved with DP. However, we can see that in this question, the order in which we choose words from wordDict \text{wordDict} wordDict is important, and a greedy strategy will not work.

Recall back in the first chapter, we said that a good way to check if a problem should be solved with DP or greedy is to first assume that it can be solved greedily, then try to think of a counterexample.

Let's say that we had s= \text{s} = s= "abcdef" and wordDict = [ \text{wordDict = [} wordDict = [ "abcde", "ef", "abc", "a", "d"] \text{]} ]. A greedy algorithm (picking the longest substring available) will not be able to determine that picking "abcde" here is the wrong decision. Likewise, a greedy algorithm (picking the shortest substring available) will not be able to determine that picking "a" first is the wrong decision.

With that being said, let's develop a DP algorithm using our framework:

For this problem, we'll look at bottom-up first.

1. An array that answers the problem for a given state

Despite this problem being unlike the ones we have seen so far, we should still stick to the ideas of the framework. In the article where we learned about multi-dimensional dynamic programming, we talked about how an index variable, usually denoted i \text{i} i is typically used in DP problems where the input is an array or string. All the problems that we have looked at up to this point reflect this.

  • With this in mind, let's use a state variable i \text{i} i, which keeps track of which index we are currently at in s \text{s} s.

  • Do we need any other state variables? The other input is wordDict \text{wordDict} wordDict - however, it says in the problem that we can reuse words from wordDict \text{wordDict} wordDict as much as we want. Therefore, a state variable isn't necessary because wordDict \text{wordDict} wordDict and what we can do with it never changes. If the problem was changed so that we can only use a word once, or say k \text{k} k times, then we would need extra state variables to know what words we are allowed to use at each state.

In all the past problems, we had a function dp \text{dp} dp return the answer to the original problem for some state. We should try to do the same thing here. The problem is asking, is it possible to create s \text{s} s by combining words in wordDict \text{wordDict} wordDict. So, let's have an array dp \text{dp} dp where dp[i] \text{dp[i]} dp[i] represents if it is possible to build the string s \text{s} s up to index i \text{i} i from wordDict \text{wordDict} wordDict. To answer the original problem, we can return dp[s.length - 1] \text{dp[s.length - 1]} dp[s.length - 1] after populating dp \text{dp} dp.

2. A recurrence relation to transition between states

At each index i \text{i} i, what criteria determines if dp[i] \text{dp[i]} dp[i] is true? First, a word from wordDict \text{wordDict} wordDict needs to be able to end at index i \text{i} i. In terms of code, this means that there is some word \text{word} word from wordDict \text{wordDict} wordDict that matches the substring of s \text{s} s that starts at index i - word.length + 1 \text{i - word.length + 1} i - word.length + 1 and ends at index i \text{i} i.

We can iterate through all states of i \text{i} i from 0 \text{0} 0 up to but not including s.length \text{s.length} s.length, and at each state, check all the words in wordDict \text{wordDict} wordDict for this criteria. For each word \text{word} word in wordDict \text{wordDict} wordDict, if s \text{s} s from index i - word.length + 1 \text{i - word.length + 1} i - word.length + 1 to i \text{i} i is equal to word \text{word} word, that means word \text{word} word ends at i \text{i} i. However, this is not the sole criteria.

Remember, we are forming s \text{s} s by adding words together. That means, if a word \text{word} word meets the first criteria and we want to use it in a solution, we would add it on top of another string. We need to make sure that the string before it is also formable. If word \text{word} word meets the first criteria, it starts at index i - word.length + 1 \text{i - word.length + 1} i - word.length + 1. The index before that is i - word.length \text{i - word.length} i - word.length, and the second criteria is that s \text{s} s up to this index is also formable from wordDict \text{wordDict} wordDict. This gives us our recurrence relation:

dp(i) = true if s.substring(i - word.length + 1, i + 1) == word and dp[i - word.length] == true for any word in wordDict, otherwise false




In summary, the criteria is:

  1. A word \text{word} word from wordDict \text{wordDict} wordDict can end at the current index i \text{i} i.

  2. If that word \text{word} word is to end at index i \text{i} i, then it starts at index i - word.length + 1 \text{i - word.length + 1} i - word.length + 1. The index before that i - word.length \text{i - word.length} i - word.length should also be formable from wordDict \text{wordDict} wordDict.

3. Base cases

The base case for this problem is another simple one. The first word used from wordDict \text{wordDict} wordDict starts at index 0 \text{0} 0, which means we would need to check dp[-1] \text{dp[-1]} dp[-1] for the second criteria, which is out of bounds. To fix this, we say that the second criteria can also be satisfied by i == word.length - 1 \text{i == word.length - 1} i == word.length - 1.
